# 题目描述

有若干个文件，使用刻录光盘的方式进行备份，假设每张光盘的容量是500MB500MB500MB，求使用光盘最少的文件分布方式所有文件的大小都是整数的MBMBMB，且不超过500MB500MB500MB;文件不能分割、分卷打包

# 输入描述

组文件大小的数据

# 输出描述

使用光盘的数量

备注

不用考虑输入数据不合法的情况:假设最多100100100个输入文件。

输入

100,500,300,200,400

输出

3

说明

(100,400)(100,400)(100,400),(200,300)(200,300)(200,300),(500)(500)(500)333张光盘即可。

输入和输出内容都不含空格。

输入

1,100,200,300

输出

2

说明

无

# 思路:贪心+二分+dfs+排序

本题成分有点多，且听我慢慢道来

首先本题可以抽象成：有n个不同重量的物品，现有一种只能装500单位容量的箱子，问装完这些物品最少需要多少个箱子（注：所有物品的重量都小于等于500）

考虑到如果直接一上来就说思路可能有些基础不牢固的童鞋可能一脸懵，固本题我们循序渐进地解决它（基础好点的同学可以利用一级缩进来定位自己想要看的）

  • 首先思考能不能直接dfs爆搜

    • 答案是最好不要，首先如果说我们dfs，则需要考虑怎么搜，一般来说这种类型的题目我们可以以不同视角来搜，若我们以箱子的视角来寻找该箱子需要哪些物品，则相应的，我们dfs时每层对应的则是每个箱子，即第i层是为了给第i号箱子寻找物品将其放入；当然我们也可以以物品的视角来寻找箱子，相应的，我们dfs时每层对应的则是每个箱子，即第i层是为了给第i号物品寻找箱子放入

    • 两种方法都是可以做到搜索的，但复杂度过不去（熟练dfs的童鞋可以思考如何避免重复搜索或剪枝之类的，这里就不展开了）

    • 为什么过不去？思考一件事：假设对于k个箱子，可以放入所有物品，且k-1个箱子也可以放入所有物品，此时我们必须承认一点，对于k个箱子我们放入所有物品的方案数是不唯一的，固我们在搜索时若已经将答案更新为k时，正在尝试搜索更小的k-1，我们将会不可避免的再次搜索出箱子数为k的情况，而事实是我已经知道k个箱子是一定可以放入所有的物品了，我们并不关心物品具体是怎么放的，我们只想尝试搜索出更少的k-1箱子数的情况；当然我们也可以考虑到一种情况是k已经是最小了，但我们的爆搜可能需要搜索出所有k的情况才能知道k-1个箱子是无法装入所有物品的这个事实。

    • 以上的问题可能有同学想到：我们不能通过剪枝提前将其余所有k的情况去除吗？但事实是物品的重量取值非常广（别小看这1~500的取值），我们如果要提前知道剩下没放入箱子的物品能否精确放入固定箱子数为k是非常困难的

    • 所以这就是为什么我们最好不要只用爆搜来找答案

  • 其次，反过来想，我们是不是只要知道了箱子数为k时能放入所有物品就行了（即爆搜时搜到一种情况就够了），我们没必要搜索出所有放的方法，可能有基础的同学一开始就想到，这题应该是迭代加深搜，那么什么是迭代加深搜？

    • 没学过的童鞋可能很迷惑，甚至没听过这个词汇，别急，就用这题来给你讲明白
    • 首先我们对于箱子数为k的情况，我们只要搜索出一种方案出来便可证明箱子数为k是可行的
    • 假设我们已经写好了一个dfs，其关键需要传入一个k值，任务便是搜索箱子数为k是否能放入所有物品，搜索出一种即可退出dfs且返回true，若一种都搜不出来则返回false（相信这种dfs的复杂度应该显然比直接搜最少箱子数要低得多吧）
    • 现在我们在dfs外面将k从小到大遍历，将其传入dfs
    • 对于第一个dfs返回为true的k即是答案

  • 现在我们再来思考如何dfs搜索

    • 为了搜索方便，我们需要以物品的视角来寻找箱子，相应的，我们dfs时每层对应的则是每个箱子，即第i层是为了给第i号物品寻找箱子放入（为什么不是箱子找物品，而是物品找箱子，主要是因为一个箱子可以放多个物品，而一个物品只会放入一个箱子，不可能说一个物品可以同时放入两个箱子）
    • 再来我们为了避免重复搜索我们需要将箱子从大到小排序，即我们需要贪心先考虑大的物品，再放小的物品（不然从小到大复杂度会比较高，这里留给读者自己思考）
    • 我们需要创建一个长度为k的数组来模拟k个箱子的存放情况
    • 剪枝：为了防止不必要的搜索，对于相同剩余容量的箱子，我们当前的物品（即当前dfs层对应的物品）需要放入编号（即下标）最大的那个箱子（具体看代码）
    • 当找到一种情况放入所有物品时（即所有物品都被dfs完了），需要结束整个dfs，并返回true

  • 最后思考能不能利用二分优化

    • 事实上本题用迭代加深搜也是可以过的
    • 但是不难发现若对于箱子数为k的情况能放入所有物品，则箱子数为k+1也一定能，即符合二分性
    • 固我们就可以不用从小到大遍历而是利用二分来找这个k了

# C++

#include <iostream>
#include <vector>
#include <algorithm>

using namespace std;

bool partition(vector<int>& buckets, vector<int>& nums, int index) {
    if (index == nums.size()) {
        return true;
    }

    int select = nums[index];
    for (int i = 0; i < buckets.size(); i++) {
        if (i > 0 && buckets[i] == buckets[i - 1]) { // 剪枝
            continue;
        }

        if (buckets[i] + select <= 500) {
            buckets[i] += select;
            if (partition(buckets, nums, index + 1)) {
                return true;
            }
            buckets[i] -= select; // 回溯
        }
    }

    return false;
}

bool check(int count, vector<int>& nums) {
    vector<int> buckets(count, 0);
    return partition(buckets, nums, 0);
}

int main() {
    vector<int> nums;
    int num;
    while (cin >> num) {
        nums.push_back(num);
        getchar();
    }

    sort(nums.rbegin(), nums.rend());

    int low = 1;
    int high = nums.size();
    int ans = high;

    while (low <= high) {
        int mid = (low + high) >> 1;
        if (check(mid, nums)) {
            ans = mid;
            high = mid - 1;
        } else {
            low = mid + 1;
        }
    }
    cout << ans << endl;
    return 0;
}

public class Main {
    public static boolean dfs(int[] buckets, int[] nums, int index) {
        if (index == -1) {
            return true;
        }

            if (i != 0 && buckets[i] == buckets[i - 1]) { // 剪枝
                continue;

            if (buckets[i] + nums[index] <= 500) {
                buckets[i] += nums[index];
                if (dfs(buckets, nums, index - 1)) {
                    return true;

                buckets[i] -= nums[index]; // 回溯
            }

        return false;
    }

# JavaScript

function dfs(buckets, nums, index) {
  if (index === nums.length) {
    return true;
  }

  for (let i = 0; i < buckets.length; i++) {
    if (i !== 0 && buckets[i] === buckets[i - 1]) { //剪枝
      continue;
    }

    if (buckets[i] + nums[index] <= 500) {
      buckets[i] += nums[index]; //回溯
      if (dfs(buckets, nums, index + 1)) {
        return true;
      }
      buckets[i] -= nums[index];
    }
  }

  return false;
}

function check(count, nums) {
  const buckets = new Array(count).fill(0);
  return dfs(buckets, nums, 0);
}

# Go

package main

import (
	"bufio"
	"fmt"
	"os"
	"sort"
	"strconv"
	"strings"
)

func dfs(buckets []int, nums []int, index int) bool {
	if index == len(nums) {
		return true
	}

	for i := range buckets {
		if i != 0 && buckets[i] == buckets[i-1] { //剪枝
			continue
		}

		if buckets[i]+nums[index] <= 500 {
			buckets[i] += nums[index] //回溯
			if dfs(buckets, nums, index+1) {
				return true
			}
			buckets[i] -= nums[index]
		}
	}

	return false
}

func check(count int, nums []int) bool {
	buckets := make([]int, count)
	return dfs(buckets, nums, 0)
}

func main() {
	reader := bufio.NewReader(os.Stdin)

	numsStr, _ := reader.ReadString('\n')
	numsStr = strings.TrimSpace(numsStr)
	numsStr = strings.ReplaceAll(numsStr, ",", " ")
	numsArr := strings.Fields(numsStr)
	var nums []int
	for _, numStr := range numsArr {
		num, _ := strconv.Atoi(numStr)
		nums = append(nums, num)
	}

	sort.Sort(sort.Reverse(sort.IntSlice(nums)))

	l := 1
	r := len(nums)
	ans := r
	// 二分模板
	for l <= r {
		mid := (l + r) / 2
		if check(mid, nums) {
			ans = mid
			r = mid - 1
		} else {
			l = mid + 1
		}
	}

	fmt.Println(ans)
}
