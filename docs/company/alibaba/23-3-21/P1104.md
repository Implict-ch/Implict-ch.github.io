### 题目内容
塔子哥有两个机器，一个是他自己的电脑，另一个是他队友黑白胖球之一$Cerq$的电脑。

他现在有一个自己的任务，执行这个任务的消耗为一个正整数 $cost$ ( $1 \leq cost \leq 2e5$ )，现在他把这个任务分成了 $k$ ( $1\le k\le cost$ )个子任务，每个子任务都是一个击杀对手或者完成目标的动作。他把这些子任务按照顺序分配给了两个机器，先让自己的电脑执行若干个子任务，然后再让$Cerq$的电脑执行剩下的子任务。这样做的目的是为了让两个机器的消耗恰好相等。

现在，请你帮忙编写一个函数，返回合法的切割方案数。由于答案可能过大你需要将答案对 $10^9+7$ 取模。

### 样例
**输入**
~~~
8 3
~~~
**输出**
~~~
6
~~~

**样例说明**

1. $(4) + (1+3)$

2. $(4) + (3+1)$

3. $(4) + (2+2)$

4. $(1+3) + (4)$

5. $(3+1) + (4)$

6. $(2+2) + (4)$

## 思路

### 组合数学

不难发现，就是将$n$分成两堆，每堆$\frac{n}{2}$个数。对于一个堆，我们将其划分成$i$个正整数相加的方案就是$C(\frac{n}{2}- 1, i - 1)$  。

​			这是因为可以看作$\frac{n}{2}$个$1$摆成一排，有$\frac{n}{2}-1$个空隙，我们选择其中的$i-1$个空隙即可分成$i$个部分了。



  所以枚举$i$,另一堆就是$k-i$ 。答案就是
$$
\sum_{i=1}^{k-1}C(\frac{n}{2}- 1, i - 1) * C(\frac{n}{2}- 1, k - i - 1)
$$
  答案比较大，需要**取模+求逆元**。

### CPP

```cpp
#include<iostream>
#include<vector>
using namespace std;

const int mod = 1000000007;
vector<int> fac(200006);
// 快速幂 求解 逆元
long long ksm(long long a, int b){
    long long ans = 1;
    long long base = a % mod;
    while(b != 0){
        if(b & 1){
            ans = ans * base % mod;
        }
        base = base * base % mod;
        b >>= 1;
    }
    return ans;
}
// 求组合数C(n , m)
int comb(int n, int m){
    if(m > n){
        return 0;
    }
    long long x = fac[n];
    long long y = fac[m] * fac[n - m] % mod;
    return x * ksm(y, mod - 2) % mod;
}

int getMethods(int cost, int k){
    // 预处理阶乘
    fac[0] = 1;
    for(int i = 1; i <= 200005; i++){
        fac[i] = fac[i-1] * i % mod; 
    }
    if(cost % 2 != 0){
        return 0;
    }
    // 根据题目公式可得
    int n = cost / 2;
    int ans = 0;
    for(int i = 1; i < k; i++){
        ans = (ans + comb(n - 1, i - 1) * comb(n - 1, k - i - 1) % mod) % mod;
    }
    return ans;
}

int main(){
    int cost, k;
    cin >> cost >> k;
    cout << getMethods(cost, k) << endl;
    return 0;
}
```

### Js

特别注意,与其他语言不同的是，js的最大整数为$2^{53}-1 \approx 9e15$  ,那就意味着$1e9+7$ 这个模数下即使是正确的取模方式，相乘操作也会导致溢出。所以所有乘法需要替换成**快速乘** 来将乘法转加法，这样就一定不会溢出。 

由于$O(1)$的乘法我们使用快速乘代替(实现类似于快速幂)，那么我们的总复杂度也需要乘上一个$log\ mod \approx 31$ 的常数。

```js
function getMethods(cost, k) {
  const mod = 1000000007;
  var fac = [1];
  
  // 封装快速乘，实现类似快速幂
  function mult(a, b) {
    let ans = 0;
    let base = a;
    while (b != 0) {
      if ((b & 1) == 1) {
        ans = (ans + base) % mod;
      }
      base = (base + base) % mod;
      b >>= 1;
    }
    return ans;
  }

  for (let i = 1; i < 200006; i++) {
    fac.push(mult(fac[i-1], i));
  }
  // 快速幂
  function ksm(a, b) {
    let ans = 1;
    let base = a;
    while (b != 0) {
      if ((b & 1) == 1) {
        ans = mult(ans, base);
      }
      base = mult(base, base);
      b >>= 1;
    }
    return ans;
  }

  if (cost % 2 !== 0) {
    return 0;
  }
  // 求组合数
  function comb(n, m) {
    if (m > n) {
      return 0;
    }
    const x = fac[n];
    const y = mult(fac[m], fac[n-m]);
    return mult(x, ksm(y, mod-2));
  }

  const n = cost / 2;
  let ans = 0;
  // 根据题解公式，求解答案
  for (let i = 1; i < k; i++) {
    ans = (ans + mult(comb(n-1, i-1), comb(n-1, k-i-1))) % mod;
  }
  return ans;
}

process.stdin.resume();
process.stdin.setEncoding('utf-8');
let input = '';
process.stdin.on('data', (data) => {
  input += data;
  return;
});
process.stdin.on('end', () => {
  const lines = input.trim().split('\n');
  const [cost, k] = lines[0].trim().split(' ').map(Number);
  console.log(getMethods(cost, k));
});
```
